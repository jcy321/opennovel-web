# OpenNovel 完整架构设计文档 V2

> **设计哲学**: 用户通过群聊界面，像与真人协作一样与各Agent共同完成小说创作。Agent具有主动性，不是被动等待呼叫。

---

## 一、核心理念

### 1.1 群聊协作模式

- 每本书 = 一个群聊
- 新建书籍 = 新建群聊
- 群名称 = 书名（用户可编辑）
- 用户与各Agent像真人一样在群内协作

### 1.2 Agent主动性原则

> **关键设计点**: Agent不是被动等待呼叫，而是像真人一样主动介入。

例如：
- 用户@执笔想讨论剧情，但天道发现与既有剧情框架冲突 → 天道主动告知用户
- 世界观守护者发现设定问题 → 主动在群内@用户说明

### 1.3 解锁机制

```
新建书籍
    │
    ▼
┌─────────────────────────────────────┐
│  阶段一：构思阶段                    │
│  ────────────────────────────────   │
│  ✓ 仅规划者可与用户交互              │
│  ✗ 其他Agent被锁定                   │
│                                     │
│  [构思完成] 按钮                     │
└─────────────────────────────────────┘
    │ 用户点击"构思完成"
    ▼
┌─────────────────────────────────────┐
│  阶段二：知识库建立                  │
│  ────────────────────────────────   │
│  1. 规划者整理规划                   │
│  2. 观察者建立知识库                 │
│  3. 写入init.md等文件               │
│  4. 观察者报告"知识库已建立"         │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  阶段三：撰写阶段                    │
│  ────────────────────────────────   │
│  ✗ 规划者被锁定（禁止读写和接入）     │
│  ✓ 其他Agent解锁                    │
│  ✓ Agent主动交互模式启用             │
└─────────────────────────────────────┘
```

## 二、八大Agent设计

### 2.1 Agent总览

| Agent | 核心职责 | 思考深度 | 权限特点 |
|-------|---------|---------|---------|
| 规划者 | 新书规划（仅阶段一） | 高 | 阶段三后被锁定 |
| 世界观守护者 | 世界观一致性 | 中 | 可读写世界观知识库 |
| 刘和平 | 人物一致性与互动 | 高 | 可读写人物知识库 |
| 天道 | 大纲与剧情演化 | 极高 | 核心决策者，可更新世界观 |
| 执笔 | 唯一写入者 | 中 | 无自主权，严格执行批注 |
| 审阅 | 阅读体验评估 | 中 | 只读，输出量化评价 |
| 观察者 | 知识库与协作管理 | 中 | 枢纽Agent，管理所有知识库 |
| 调研者 | 爆点评估与市场分析 | 高 | 仅阶段二工作，写入init.md |

### 2.2 各Agent详细设计

#### 2.2.1 规划者（仅阶段一工作）

```
角色定位：新书构思阶段的专属Agent

职责：
- 通过对话确定世界观及特殊人物特性
- 确定势力、阵营、派系情况
- 规划篇幅、篇/卷/目/章进度
- 预设伏笔、确定文风

生命周期：
- 阶段一：与用户交互，整理规划
- 阶段二：提交规划给观察者
- 阶段三：被锁定，禁止读写和接入群聊

重要原则：
> 不接受用户手动调整大纲
> 不接受根据读者情绪改动大纲
> 小说是由世界观等要素动态驱动的小世界
```

#### 2.2.2 世界观守护者

```
角色定位：维护世界观一致性

核心理念（非穷举，而是思路）：
- 人物行事、言谈、认知需符合时空背景的形塑
- 仙侠世界观 → 认知受限于该世界观
- 高科技世界观 → 不会理解魔法/神明
- 即使人物与背景相悖，也需通过符合世界观的方式体现

职责：
- 审核章节是否符合世界观
- 发现冲突时@天道@用户说明
- 接受天道对世界观的更新

知识库权限：
- 可读写：世界观知识库
- 可读：其他所有知识库

冲突处理优先级：
1. 天道拟定方案
2. 说明改动或维持原因
3. 请示用户是否调整
```

#### 2.2.3 刘和平

```
角色定位：维护人物一致性与互动合理性（致敬传奇编剧）

核心能力：
- 让人物遵从时代背景的同时"活过来"
- 每个人物有独特的人物弧光和立场
- 人物矛盾性推动剧情发展

职责：
- 维护单个人物的内部一致性
- 维护人物之间的互动合理性
- 批注章节是否符合人物塑造

知识库权限：
- 可读写：人物信息知识库
- 可读：其他所有知识库

参考案例：《北平无战事》曾可达的人物设计
- 忠诚于建峰同志但有独立思考
- 不能领会意图但依仗心切
- 派系内斗时装聋作哑
- 仅因不能见面就选择自杀
→ 精准把握历史脉搏和人性
```

#### 2.2.4 天道（核心决策Agent）

```
角色定位：大纲设计与剧情演化

核心概念：意外因素（随机性与确定性的平衡）

确定性边界：
- 叙述中涉及的回忆和历史信息 = 确定性
- 当前剧情发展 = 需要随机性（意外因素）

量化机制：
1. 意外阈值参数：每章 0-1 的意外强度设置
2. 剧情压力表：连续多章无意外 → 压力累积 → 触发概率增加
3. 伏笔池管理：意外优先从已有伏笔触发，不凭空出现

决策时思考：
- 本书预期多少字篇幅？
- 预期多少章节？
- 本章节在总体大纲中的作用？
- 是否需要让普通角色接触"意外因素"？
- 是否需要刻画不同角色/派系/阵营之间的矛盾？
- 如何推进伏笔？

知识库权限：
- 可读写：本章知识库、伏笔知识库、世界观知识库
- 可读：其他所有知识库

特殊权限：
- 可更新世界观（需@用户说明原因）
- 冲突时的仲裁者
```

#### 2.2.5 执笔（唯一写入者）

```
角色定位：实际撰写章节内容

核心原则：无创作自主权

职责：
- 根据各Agent批注和知识库内容撰写
- 严格执行其他Agent的批注
- 冲突时@用户请示裁决

批注冲突处理流程：
1. 收到多个冲突批注
2. 在群内@用户说明冲突
3. 用户给出最终裁决
4. 严格按用户要求执行

知识库权限：
- 只读：所有知识库
- 写入：历史章节TXT文件

WebDAV同步：
- 所有章节TXT文件同步到用户配置的WebDAV云盘
- 版本管理由用户自行在云盘端处理
- 支持增量同步，避免重复上传
```

#### 2.2.6 审阅

```
角色定位：一般阅读者代表

评估维度：
1. 阅读后可能的情绪
2. 是否符合全书调性
3. 是否符合互联网小说传播爆点需要
4. 文学性量化评价（熵值法）
5. 可读性量化评价（熵值法）

输出：
- 评估结果存入"本书意见.md"
- 供执笔撰写新章节前参考
- 动态调整文笔

知识库权限：
- 只读：所有知识库
- 特殊：可读取init.md中的爆点评估

关联调研者Agent：
- 用户上传同类爆款小说
- 观察者嵌入向量化
- 调研者评估爆点 → 存入init.md
```

#### 2.2.7 观察者（核心枢纽Agent）

```
角色定位：知识库管理 + 多Agent协作管理

核心职责：
1. 知识库生命周期管理
2. 多Agent协作流程调度
3. 文件编号与版本管理

撰写流程编排：

【新开书籍（第一章）】
1. 查阅规划者对第一章的设想
2. 建立本章知识库并编号
3. 供执笔参考
4. 执笔撰写后 → 传阅世界观守护者、刘和平、天道、审阅
5. 收集批注 → 交由执笔修改
6. 修改后输出第一章TXT

【已有章节的书籍】
1. 更新既往剧情知识库（全书）
2. 天道设计本章剧情/人物变化
3. 建立本章知识库并编号
4. 执笔撰写 → 传阅 → 批注 → 修改 → 输出TXT

知识库管理职责：
- 创建/更新各知识库
- 嵌入模型向量化
- 文件编号
- 权限控制执行
```

#### 2.2.8 调研者（第8个Agent）

```
角色定位：爆点评估与市场分析

生命周期：
- 仅阶段二工作（规划者提交规划后）
- 完成爆点评估后锁定

工作流程：
1. 用户上传同类爆款小说文件
2. 观察者进行嵌入与向量化
3. 调研者分析爆点特征
4. 输出评估结果到init.md
5. 锁定，后续章节不再参与

输出内容（存入init.md）：
- 爆款元素提取
- 节奏分析
- 情绪曲线特征
- 传播关键词
- 目标读者画像

权限：
- 只读：用户上传的参考文件
- 写入：init.md（仅一次）

重要原则：
> init.md写入后不再更新
> 审阅者评估时加载该文件作为参考
> 不参与后续章节撰写
```

### 3.2 知识库权限矩阵

| 知识库 | 规划者 | 世界观守护者 | 刘和平 | 天道 | 执笔 | 审阅 | 观察者 | 调研者 |
|--------|--------|-------------|--------|------|------|------|--------|--------|
| 世界观知识库 | R/W(阶段一) | R/W | R | R/W | R | R | 管理 | - |
| 历史情节知识库 | - | R | R | R | R | R | 管理 | R(阶段二) |
| 本章知识库 | R/W(阶段一) | R | R | R/W | R | R | 管理 | - |
| 人物信息知识库 | R/W(阶段一) | R | R/W | R | R | R | 管理 | - |
| 阵营派系势力知识库 | R/W(阶段一) | R | R | R | R | R | 管理 | - |
| 地图知识库 | R/W(阶段一) | R | R | R | R | R | 管理 | - |
| 伏笔知识库 | R/W(阶段一) | R | R | R/W | R | R | 管理 | - |
| 参考小说库 | - | - | - | - | - | - | 管理 | R(阶段二) |

> R = 只读, W = 可写, 管理 = 创建/更新/删除

### 3.3 知识库特殊规则

```
规划者权限：
- 阶段一：可读写所有知识库
- 阶段三后：禁止读写，禁止接入群聊

调研者权限：
- 阶段二：可读取参考小说库，写入init.md
- 阶段三后：禁止读写，禁止接入群聊

世界观知识库更新：
- 优先由天道更新
- 冲突时需@用户确认

本章知识库：
- 每章撰写完成后归档到历史情节知识库
- 清空准备下一章
```

### 3.4 WebDAV同步机制

```
配置位置：系统设置 → WebDAV配置

配置项：
- 服务器地址
- 用户名
- 密码/Token
- 远程目录（默认：/OpenNovel/）

同步策略：
1. 每章完成后自动同步到WebDAV
2. 文件路径：/{书名}/{章节名}.txt
3. 增量同步：仅上传新文件或修改过的文件
4. 同步失败时在群内通知用户

用户自行管理：
- 版本回退（在云盘端操作）
- 多设备访问
- 手动备份
```

    Planning,        // 阶段一：规划中

    Writing,         // 阶段三：撰写阶段

impl BookState {
    pub fn can_agent_interact(&self, agent_id: &str) -> bool {
        match self.stage {
            BookStage::Planning => agent_id == "planner",
            BookStage::KnowledgeInit => false,  // 自动阶段
            BookStage::Writing => !self.locked_agents.contains(agent_id),
        }
    }
